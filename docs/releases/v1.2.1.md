# CSS Loader for Millennium v1.2.1

Version 1.2.1 moves the direct-injection runtime mailbox into the separately
installed CSS Loader Companion plugin.

## Changes

- Publishes `build-report.json` and `runtime-state.json` to
  `%STEAM%\millennium\plugins\css-loader-companion\runtime`.
- Detects when the separately distributed CSS Loader Companion plugin is missing
  and displays a direct link to its latest GitHub release.
- Rechecks for a newly installed companion, restarts the bundled backend, and
  confirms that the direct runtime mailbox was generated before asking the user
  to restart Steam.
- Keeps the required-companion setup prompt open until the companion and runtime
  are ready, preventing an incomplete first-run setup from being dismissed.
- Makes the backend tray **Exit** action close the desktop app before stopping
  the backend, preventing the desktop health check from undoing an intentional
  exit. The packaged backend terminates directly after cleanup so the action is
  not dependent on asyncio propagating `SystemExit`.
- Adds **Restart CSS Loader** above Exit in the tray menu. It closes and
  relaunches the desktop app with a one-time request that replaces the backend.
- Stops generating a metadata-only Millennium theme and `skin.json`.
- Removes the obsolete `%STEAM%\millennium\themes\CSS Loader` mailbox after a
  successful publish.
- Refuses to recreate a partial companion directory when the plugin is not
  installed.

Install CSS Loader Companion 1.2.0 separately before opening the desktop app.
Themes, profiles, settings, and the selected Millennium theme remain unchanged.
